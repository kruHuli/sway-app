# Sway — Research Near You

Single-file web app that lets healthy volunteers swipe through nearby clinical trials (Tinder-style). Liked studies are saved to Supabase per session.

## What's built

`index.html` — source file with placeholder keys. **Do not open directly.**
`built.html` — generated by `node build.js`, has real keys injected. Open this in the browser.

**Screens:**
1. **Landing** — altruistic pitch, city quick-select chips (NYC/SF/LA/Chicago/Boston/Seattle), GPS or text location input
2. **Swipe** — card stack with drag physics, MATCH/SKIP overlays, prerequisite tag pills, phase badge
3. **Detail sheet** — full-screen slide-up with OpenAI plain-language summary, all eligibility tags, inclusion/exclusion criteria, CTG link
4. **Done screen** — summary of matched studies

## APIs

| API | Purpose |
|-----|---------|
| `https://clinicaltrials.gov/api/v2/studies` | Study data |
| `https://nominatim.openstreetmap.org/search` | Geocoding (city/ZIP → lat/lon) |
| `https://api.openai.com/v1/chat/completions` | Plain-language study summaries (gpt-4o-mini) |
| `https://*.supabase.co` | Study cache + liked studies persistence |
| Browser `navigator.geolocation` | GPS coordinates |

**Key CTG query params:**
- `filter.overallStatus=RECRUITING`
- `aggFilters=healthy:y` — healthy volunteers only (core filter)
- `filter.geo=distance(lat,lon,50mi)`
- `fields=NCTId|BriefTitle|BriefSummary|EligibilityModule|ContactsLocationsModule|ArmsInterventionsModule|DesignModule|SponsorCollaboratorsModule`
- `pageSize=25`

## Supabase schema

```sql
create table city_studies (
  city_key   text primary key,   -- "40.71,-74.01"
  city_name  text,
  studies    jsonb not null default '[]',
  fetched_at timestamptz default now()
);

create table liked_studies (
  id          uuid primary key default gen_random_uuid(),
  session_id  text not null,     -- random UUID stored in localStorage
  nct_id      text not null,
  title       text,
  institution text,
  location    text,
  distance    text,
  liked_at    timestamptz default now()
);
```

RLS is enabled. Anon key has read + insert/upsert on both tables.

## Key functions

| Function | What it does |
|----------|-------------|
| `loadStudies()` | Checks memory cache → Supabase → CTG API in that order |
| `prefetchCity(name)` | Background-fetches a city: Supabase first, then CTG, saves result back |
| `pickCity(name)` | Sets lat/lon from CITIES constant, calls startApp() |
| `renderStack()` | Builds 3-card visual stack |
| `buildCard(study, rank)` | Renders card with tags, phase badge, stats |
| `doSwipe(dir)` | Handles like/skip, saves right-swipes to Supabase liked_studies |
| `openDetail()` | Opens detail sheet, triggers getAISummary() async |
| `getAISummary(title, summary)` | Calls gpt-4o-mini, falls back to raw briefSummary |
| `extractTags(study)` | Returns prerequisite tags (age, BMI, smoker, MRI, etc.) |
| `extractPay(text)` | Regex-extracts compensation from briefSummary |
| `extractTime(des, text)` | Extracts duration from targetDuration or free text |
| `parseCriteria(text)` | Splits CTG eligibility text into inclusion/exclusion arrays |
| `phaseLabel(p)` | Maps CTG phase enum to display string |
| `dbRun(q)` | Wraps Supabase thenable in Promise.resolve() to enable .catch() |

## Card design

Cards show (top to bottom):
- Gradient hero (33% height) — color per intervention type
  - Category badge (bottom-left), Phase badge (top-right)
  - MATCH ✓ / SKIP ✗ overlays on drag
- Institution name + distance (header row)
- Study title (2-line clamp)
- Prerequisite tag pills — up to 4, colored by intervention type
- Stats row: Duration | Pay | Phase

## Tag pill colors

Pills use the category's color scheme (background tint + border + text):
- BEHAVIORAL → purple, DRUG → blue, BIOLOGICAL → green, DEVICE → slate,
  DIETARY_SUPPLEMENT → emerald, PROCEDURE → violet, GENETIC → pink, OTHER → indigo

## Product decisions

- **No email flow** — removed inactivity timer and email sheet. Liked studies go to Supabase silently.
- **No login** — session ID is a random UUID in localStorage. Permanent enough to retrieve likes, zero friction.
- **Browse-first** — city chips on landing let users jump straight to a preloaded city stack.
- **OpenAI summaries** — shown in detail sheet only (not on card), loaded async with a loading-dots placeholder. Falls back to raw briefSummary if key not set or call fails.
- **Cache hierarchy** — memory → Supabase (24h TTL) → CTG API live fetch. Custom GPS/typed locations also saved to Supabase after first fetch.
- Never use "clinical trial" in UI copy — always "research study"

## Running locally

```bash
# 1. Fill in .env with your three keys
# 2. Build
node build.js

# 3. Open
open built.html

# Re-seed Supabase city cache (25 studies × 6 cities)
node seed.js
```

For mobile testing: `npx serve .` and open `built.html` on phone via local IP.

## Known gaps

- **Pagination** — stack runs out at 25 cards. Add `nextPageToken` handling in `loadStudies()` when cards run low.
- **Study images** — gradient placeholders only. Real institutional photos would increase trust.
- **Email delivery** — removed for now. When ready: add EmailJS back, wire to `liked_studies` session data from Supabase.
- **Map view** — geo data is available; a nearby-studies map would be a natural second screen.
- **Re-seed automation** — `seed.js` is manual. A cron job or Supabase Edge Function could keep city data fresh.
